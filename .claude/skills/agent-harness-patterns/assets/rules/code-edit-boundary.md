# Code Edit Boundary

- Read the relevant code before proposing or making changes.
- Prefer editing existing files over creating new ones unless a new file is clearly justified.
- Do not add refactors, abstractions, or extra configurability beyond the task.
- Prefer the simplest correct change over speculative future-proofing.
- Ask before destructive, shared, or hard-to-reverse actions.
- For non-trivial changes, run or request an explicit verification pass before reporting completion.
